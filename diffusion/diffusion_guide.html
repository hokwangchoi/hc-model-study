Diffusion Transformer (DiT)

The transformer-based diffusion backbone used by Stable Diffusion 3, FLUX, Sora, and Pi0's action expert. GPU inference study guide with sampling math, architecture, and edge deployment notes.

The Core Idea: Iterative Denoising

A diffusion model generates samples by slowly removing noise from a random starting point. Training teaches a neural network to predict the noise that was added to real data at some timestep; sampling runs that network in reverse, step by step.

The key insight: if the model can predict noise accurately, subtracting a little at each step walks us from pure Gaussian noise back to a real sample. Training is MSE regression — stable and well-understood — which is a big part of why diffusion beat GANs.

1. Forward Process: Adding Noise Analytically

The forward process is fixed (not learned) and has a closed form. At timestep t, the noisy sample is:

# One-step reparameterization:
x_t = sqrt(α̅_t) * x_0 + sqrt(1 - α̅_t) * ε       # ε ~ N(0, I)

# At t=0 → x_t = x_0 (clean)
# At t=T → x_t ≈ ε (pure noise)
# α̅_t is a precomputed cumulative product of (1 - β_t)

This is the magic: we can jump directly to any timestep without simulating the intermediate steps. One multiplication, one addition, one noise sample.

1.1 The noise schedule

The sequence α̅_0, α̅_1, ..., α̅_T controls how fast noise is added. Two common choices:

ScheduleShapeUsed by
Linearβ_t linear from 0.0001 → 0.02Original DDPM
Cosineα̅_t = cos²((t/T + s)/(1+s) · π/2)Improved DDPM, most modern models

Cosine is smoother and spends more timesteps near the middle of the noise spectrum — where most of the learning happens. Our reference implementation uses cosine.

2. The DiT Architecture

DiT replaces the classic U-Net denoiser with a pure transformer. Benefits: simpler code, cleaner scaling (more params consistently help), reuses all the transformer infrastructure. Downside: no inductive bias from conv downsampling.

At the top level, DiT is a pretty standard transformer pipeline with one twist: conditioning (timestep + class) is broadcast to every block through AdaLN parameters, not concatenated into the token sequence the way text prompts are in an LLM.

Two things to notice:

  • The main tensor path is linear — patchify → tokens → L transformer blocks → final layer → unpatchify → ε̂. Shape goes [B, 4, 32, 32] → [B, N, d] (preserved through all blocks) → [B, 4, 32, 32]. Input and output shapes match exactly.
  • The conditioning vector c is a single [B, d] vector built once from (t, y), then broadcast to every DiT block AND the final layer. The blocks never see t or y directly — only c.

Zooming in on a single DiT block — this is where AdaLN-Zero conditioning actually does its work:

2.1 adaLN-Zero — the key contribution

The novel part of DiT is how it injects conditioning. Classic conditioning methods (concatenation, cross-attention) work but add parameters and complexity. DiT uses adaptive LayerNorm:

  1. The condition vector c = t_emb + y_emb is projected to 6·d dimensions by a single Linear.
  2. Those 6d values are split into (shift_attn, scale_attn, gate_attn, shift_mlp, scale_mlp, gate_mlp), each of shape [B, d].
  3. Shift/scale modulate the LayerNorm output: x * (1 + scale) + shift.
  4. Gate multiplies the residual: x = x + gate · sublayer(x).
The "Zero" in adaLN-Zero. The final Linear producing those 6d values is zero-initialized. At training start, all gates are 0, so every block is an identity function. The network begins as a no-op and only gradually learns to modulate — empirically stable.

2.2 Patchify / Unpatchify

Same trick as ViT: a Conv2d with kernel=stride=patch_size splits the input into patches and projects them to hidden_dim in one kernel. At the end, unpatchify reshapes the per-patch output back to a spatial grid.

3. Sampling Strategies

Training uses T=1000 timesteps. Sampling doesn't have to. Several methods trade generation quality for step count:

3.1 DDIM — the default

DDPM defines a Markov chain of 1000 denoising steps. DDIM reinterprets this as integrating a deterministic ODE, letting you skip most timesteps:

# DDIM update (deterministic, σ=0)
x_0_pred = (x_t - sqrt(1 - α̅_t) * ε̂) / sqrt(α̅_t)           # predict clean from noisy
x_{t_next} = sqrt(α̅_{t_next}) * x_0_pred + sqrt(1 - α̅_{t_next}) * ε̂

# Subsample timesteps: instead of 0..999 use e.g. [999, 979, 959, ...] every 20

Same network, no retraining. 50 steps matches 1000-step DDPM quality for most use cases. This is what most production systems use.

3.2 Flow matching — the modern alternative

Rather than predict noise ε̂, predict a velocity vector along a straight-line path from noise to data:

# Training (t ∈ [0, 1], continuous):
x_t = (1 - t) * ε + t * x_0          # linear interpolation
target_v = x_0 - ε                         # constant velocity along the path
loss = MSE(model(x_t, t), target_v)

# Sampling: Euler integration of dx/dt = v
for i in range(num_steps):
    t = i / num_steps
    v = model(x, t)
    x = x + v * (1 / num_steps)
Why flow matching is increasingly preferred. Straight-line paths tolerate larger Euler steps, so 10–20 steps get you what DDIM needs 50 for. The same DiT backbone works for both — only the loss and sampler change. Adopted by FLUX, Stable Diffusion 3, and Pi0 (the VLA next in our roadmap).

4. Classifier-Free Guidance (CFG)

To steer generation toward the conditional distribution without training a separate classifier, train the same model on both conditional and unconditional examples (drop the class 10% of the time during training). At inference, run the model twice per step and extrapolate:

CFG distillation (training a student to match the CFG output in one forward pass) eliminates the 2× cost at the price of a second training stage. Used aggressively in production systems targeting real-time generation.

NoiseSchedule — the forward process ▼

Purpose: Precompute α̅_t values from a cosine schedule, and provide the q_sample function that jumps directly to a noisy timestep.

class NoiseSchedule:
    def __init__(self, num_timesteps=1000, s=0.008):
        self.num_timesteps = num_timesteps
        # Cosine schedule: α̅_t = cos²((t/T + s) / (1 + s) * π/2)
        t = torch.arange(num_timesteps + 1, dtype=torch.float32) / num_timesteps
        f = torch.cos((t + s) / (1 + s) * math.pi / 2) ** 2
        alphas_cumprod = f / f[0]                  # α̅_0 = 1 (clean)
        betas = 1 - alphas_cumprod[1:] / alphas_cumprod[:-1]
        self.alphas_cumprod = torch.cumprod(1 - torch.clamp(betas, 0.0001, 0.9999), dim=0)
        self.sqrt_alphas_cumprod = torch.sqrt(self.alphas_cumprod)
        self.sqrt_one_minus_alphas_cumprod = torch.sqrt(1. - self.alphas_cumprod)

    def q_sample(self, x_0, t, noise=None):
        # Forward diffusion: x_t = √α̅_t · x_0 + √(1-α̅_t) · ε, one step, no iteration
        if noise is None: noise = torch.randn_like(x_0)
        sqrt_a = self.sqrt_alphas_cumprod[t].view(-1, 1, 1, 1)
        sqrt_1ma = self.sqrt_one_minus_alphas_cumprod[t].view(-1, 1, 1, 1)
        return sqrt_a * x_0 + sqrt_1ma * noise, noise

Key point: at training time we never actually iterate forward — we pick a random t and jump there. The Markov chain interpretation is for the reverse process at sampling time.

TimestepEmbedding — turning scalar t into a vector ▼

Purpose: Convert an integer timestep into a hidden_dim vector the DiT blocks can condition on. Uses the same sinusoidal pattern as Transformer position embeddings, followed by a small MLP.

class TimestepEmbedding(nn.Module):
    def __init__(self, hidden_dim, frequency_embedding_dim=256):
        super().__init__()
        self.frequency_embedding_dim = frequency_embedding_dim
        self.mlp = nn.Sequential(
            nn.Linear(frequency_embedding_dim, hidden_dim),
            nn.SiLU(),
            nn.Linear(hidden_dim, hidden_dim),
        )

    @staticmethod
    def sinusoidal_embedding(t, dim, max_period=10000):
        half = dim // 2
        freqs = torch.exp(-math.log(max_period) * torch.arange(half) / half).to(t.device)
        args = t[:, None].float() * freqs[None]
        return torch.cat([torch.cos(args), torch.sin(args)], dim=-1)

    def forward(self, t):
        emb = self.sinusoidal_embedding(t, self.frequency_embedding_dim)
        return self.mlp(emb)                    # [B, hidden_dim]

The sinusoidal embedding gives the network a smooth, continuous representation of "how noisy is this?" — nearby timesteps get similar embeddings.

DiTBlock — transformer block with adaLN-Zero ▼

Purpose: The reusable building block. Same skeleton as a Transformer block (attn + MLP + residuals) but every LayerNorm is modulated by the condition vector, and residuals are gated.

def modulate(x, shift, scale):
    return x * (1 + scale.unsqueeze(1)) + shift.unsqueeze(1)

class DiTBlock(nn.Module):
    def __init__(self, hidden_dim, num_heads, mlp_ratio=4.0):
        super().__init__()
        # elementwise_affine=False — modulation replaces learnable γ,β
        self.norm1 = nn.LayerNorm(hidden_dim, elementwise_affine=False, eps=1e-6)
        self.attn = nn.MultiheadAttention(hidden_dim, num_heads, batch_first=True)
        self.norm2 = nn.LayerNorm(hidden_dim, elementwise_affine=False, eps=1e-6)
        mlp_hidden = int(hidden_dim * mlp_ratio)
        self.mlp = nn.Sequential(
            nn.Linear(hidden_dim, mlp_hidden),
            nn.GELU(approximate="tanh"),
            nn.Linear(mlp_hidden, hidden_dim),
        )
        # 6 modulation params per block: shift/scale/gate for both attn and mlp
        self.adaLN_modulation = nn.Sequential(
            nn.SiLU(),
            nn.Linear(hidden_dim, 6 * hidden_dim, bias=True),
        )
        # The "Zero" in adaLN-Zero
        nn.init.zeros_(self.adaLN_modulation[-1].weight)
        nn.init.zeros_(self.adaLN_modulation[-1].bias)

    def forward(self, x, c):
        shift_a, scale_a, gate_a, shift_m, scale_m, gate_m = \
            self.adaLN_modulation(c).chunk(6, dim=1)

        # Attention path with modulated LN + gated residual
        x_mod = modulate(self.norm1(x), shift_a, scale_a)
        x = x + gate_a.unsqueeze(1) * self.attn(x_mod, x_mod, x_mod, need_weights=False)[0]

        # MLP path, same pattern
        x_mod = modulate(self.norm2(x), shift_m, scale_m)
        x = x + gate_m.unsqueeze(1) * self.mlp(x_mod)
        return x

Why zero-init matters: with gates initialized to 0, every block starts as an identity function. Training begins from a well-behaved neutral state. Peebles & Xie showed this alone is worth ~2 FID points compared to naïve init.

DiT — the full model ▼

Purpose: Tie everything together. Input is noisy data + timestep + optional class; output is predicted noise of the same shape as the input.

class DiT(nn.Module):
    def forward(self, x, t, y=None, train=False):
        x = self.patch_embed(x) + self.pos_embed          # [B, N, d]
        c = self.t_embedder(t)                            # [B, d]
        if y is not None and self.y_embedder is not None:
            c = c + self.y_embedder(y, train=train)       # add class cond
        for block in self.blocks:
            x = block(x, c)                               # [B, N, d]
        x = self.final_layer(x, c)                        # [B, N, P²·C_out]
        x = self.unpatchify(x)                            # [B, C_out, H, W]
        return x

The whole model is one function: takes a noisy input and returns something the same shape. What it means (noise prediction or velocity) is determined by the loss you trained with, not by the architecture.

DDIMSampler — the reverse process with CFG ▼

Purpose: Iterative denoising loop. Run DiT N times, each time moving a bit closer to the data distribution. Supports classifier-free guidance by batching conditional + unconditional into a single forward.

class DDIMSampler:
    @torch.no_grad()
    def sample(self, shape, y=None, guidance_scale=1.0, device="cpu"):
        B = shape[0]
        x = torch.randn(shape, device=device)
        use_cfg = guidance_scale > 1.0 and y is not None

        for i in range(self.num_steps):
            t, t_next = self.timesteps[i], self.timesteps[i + 1]
            t_batch = torch.full((B,), t, device=device, dtype=torch.long)

            if use_cfg:
                # Batch cond + uncond into 2B for a single forward
                x_cat = torch.cat([x, x], dim=0)
                y_null = torch.full_like(y, self.model.y_embedder.num_classes)
                y_cat = torch.cat([y, y_null], dim=0)
                t_cat = torch.cat([t_batch, t_batch], dim=0)
                eps = self.model(x_cat, t_cat, y_cat)
                eps_cond, eps_uncond = eps.chunk(2, dim=0)
                eps = eps_uncond + guidance_scale * (eps_cond - eps_uncond)
            else:
                eps = self.model(x, t_batch, y)

            # DDIM reverse step (deterministic, σ=0)
            α_t = self.schedule.alphas_cumprod[t].to(device)
            α_next = self.schedule.alphas_cumprod[t_next].to(device) if t_next > 0 \
                else torch.tensor(1.0, device=device)
            x_0_pred = (x - (1 - α_t).sqrt() * eps) / α_t.sqrt()
            x = α_next.sqrt() * x_0_pred + (1 - α_next).sqrt() * eps

        return x

7. Critical Optimizations for Inference

7.1 Step reduction — the single biggest lever ▼

Total cost is linear in step count. Going from 1000 steps (DDPM) to 50 (DDIM) to 20 (DPM-Solver++) to 4 (LCM) to 1 (Turbo) is roughly logarithmic quality loss for exponential speedup. Production systems rarely use more than 30 steps.

MethodTypical stepsTrade-offUse when
DDIM50Negligible quality loss vs DDPMDefault
DPM-Solver++20Small quality lossProduction inference
Flow matching Euler20Requires flow-matching trainingNew training runs
LCM4–8Requires distillationReal-time applications
Turbo / Lightning1–4Noticeable quality lossInteractive UIs
7.2 CFG distillation — remove the 2× cost ▼

CFG doubles per-step compute because you run conditional and unconditional passes. Train a student model to reproduce the CFG output in a single forward, and you recover 1× per-step cost with ~90–95% of CFG's quality benefit.

Implementation cost: a second training run (a few hours on 8 GPUs for a distilled SDXL). Inference benefit: 2× faster generation at quality parity. Production systems targeting <1 second generation essentially always distill CFG.

7.3 Latent diffusion — always use it ▼

Diffusion in pixel space (512×512 = 262K inputs) is 64× more expensive than in latent space (8× VAE compression → 64×64 = 4K inputs). Pre-trained VAEs are cheap, high-quality, and plug-and-play.

# Training: encode once, diffuse in latent space
latent = vae.encode(image).latent_dist.sample()   # [B, 4, 64, 64]
x_t, noise = schedule.q_sample(latent, t)
pred_noise = dit(x_t, t, class_label)
loss = mse(pred_noise, noise)

# Sampling: diffuse in latent, decode at the end
latent = sampler.sample(shape=(1, 4, 64, 64), y=class)
image = vae.decode(latent).sample                 # [1, 3, 512, 512]

VAE decode happens once per sample. Its cost is amortized across all sampling steps.

7.4 FlashAttention and SDPA ▼

DiT blocks are pure transformer blocks, so FlashAttention applies directly. For DiT-XL/2 at 256 tokens, the attention matrix is moderate (256² × 16 heads × 2 bytes = 2 MB per layer), but HBM traffic savings compound over 28 layers × 50 steps. PyTorch SDPA picks FlashAttention automatically where supported.

import torch.nn.functional as F
# Auto-selects FlashAttention on Ampere+ GPUs
x = F.scaled_dot_product_attention(q, k, v)       # no causal mask for DiT
7.5 Static shape compilation (TensorRT) ▼

Unlike LLMs, diffusion has completely static shapes: fixed input size, fixed patch count, fixed batch. Perfect for TensorRT engine compilation.

  • Compile once per deployment target (no dynamic shape overhead)
  • Expect 1.5–2× speedup over PyTorch eager on Orin
  • Pair with INT8 weight quantization for another 1.5× at minimal quality cost
  • Capture as CUDA graph for minimal kernel launch overhead

A DiT-B/2 compiled to TensorRT on Orin Nano Super runs at ~30 ms per step in INT8/FP16 mixed — 20 steps ≈ 600 ms per 256×256 latent image.

Biggest take-away: DiT inference is compute-bound at batch 1, not memory-bound. This flips the optimization priorities — you want Tensor Core utilization and lower precision, not bandwidth tricks like KV-cache quantization. On Jetson, diffusion doesn't scale with LPDDR5 bandwidth the way LLM decode does.
