1 Notation & Tensor Shapes

📥
Input Embeddings + Positional Encoding
Input: [B, N] → Output: [B, N, d]
Memory-bound

What it does

Converts discrete token IDs into continuous vector representations and adds positional information since transformers have no inherent notion of order.

Mathematical formula
X = Embedding(tokens) + PositionalEncoding(positions)
Embedding: Lookup table of shape [V, d]
Positional encoding: Sinusoidal or learned, shape [N_max, d]

PyTorch implementation

Python PyTorch docs → 
import torch
import torch.nn as nn
import math

class TransformerEmbedding(nn.Module):
    def __init__(self, vocab_size, d_model, max_seq_len, dropout=0.1):
        super().__init__()
        self.token_emb = nn.Embedding(vocab_size, d_model)  # [V, d]
        self.pos_emb = nn.Embedding(max_seq_len, d_model)   # [N_max, d]
        self.dropout = nn.Dropout(dropout)
        self.d_model = d_model

    def forward(self, x):
        # x: [B, N] token indices
        B, N = x.shape
        positions = torch.arange(N, device=x.device).unsqueeze(0)  # [1, N]

        # Scale embeddings by sqrt(d_model) as in original paper
        tok = self.token_emb(x) * math.sqrt(self.d_model)  # [B, N, d]
        pos = self.pos_emb(positions)                       # [1, N, d]

        return self.dropout(tok + pos)  # [B, N, d]

GPU kernel considerations

Embedding lookup is a gather operation — essentially random memory access. This is memory-bound on GPU. For large vocabularies, consider:

  • Fused embedding + position addition
  • Quantized embeddings (INT8)
  • Tensor parallelism across vocabulary dimension
🎯
Multi-Head Attention
Input: [B, N, d] → Output: [B, N, d]
Compute + Memory

What it does

Allows each position to attend to all other positions in the sequence. Multiple "heads" learn different attention patterns in parallel.

Scaled dot-product attention
Attention(Q, K, V) = softmax(QKᵀ / √d_k) × V
Q, K, V: Query, Key, Value matrices — linear projections of input
√d_k: Scaling factor to prevent dot products from growing too large
Multi-head formulation
MultiHead(X) = Concat(head₁, ..., headₕ) × Wᵒ
headᵢ: Attention(X×Wᵢᵠ, X×Wᵢᴷ, X×Wᵢⱽ)
Wᵒ: Output projection [h×d_v, d]

PyTorch implementation

Python PyTorch docs → 
class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads, dropout=0.1):
        super().__init__()
        assert d_model % n_heads == 0

        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads

        # Q, K, V projections (often fused into single weight)
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model, bias=False)

        self.dropout = nn.Dropout(dropout)
        self.scale = self.d_k ** -0.5

    def forward(self, x, mask=None):
        B, N, _ = x.shape

        # Project to Q, K, V: [B, N, d] → [B, N, d]
        Q = self.W_q(x)  # GEMM: [B, N, d] × [d, d]
        K = self.W_k(x)  # GEMM: [B, N, d] × [d, d]
        V = self.W_v(x)  # GEMM: [B, N, d] × [d, d]

        # Reshape for multi-head: [B, N, d] → [B, h, N, d_k]
        Q = Q.view(B, N, self.n_heads, self.d_k).transpose(1, 2)
        K = K.view(B, N, self.n_heads, self.d_k).transpose(1, 2)
        V = V.view(B, N, self.n_heads, self.d_k).transpose(1, 2)

        # Attention scores: [B, h, N, d_k] × [B, h, d_k, N] → [B, h, N, N]
        scores = torch.matmul(Q, K.transpose(-2, -1)) * self.scale

        # Apply causal mask if provided
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))

        # Softmax over last dimension (memory-bound!)
        attn_weights = torch.softmax(scores, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # Apply attention to values: [B, h, N, N] × [B, h, N, d_k] → [B, h, N, d_k]
        context = torch.matmul(attn_weights, V)

        # Reshape back: [B, h, N, d_k] → [B, N, d]
        context = context.transpose(1, 2).contiguous().view(B, N, self.d_model)

        # Output projection
        return self.W_o(context)  # GEMM: [B, N, d] × [d, d]

Operation breakdown

Operation FLOPs formula Memory access Kernel
QKV projection 6 × B × N × d² O(d²) weights cuBLAS GEMM
QKᵀ matmul 2 × B × h × N² × d_k O(N²) output! cuBLAS batched GEMM
Scale B × h × N² Elementwise Fused kernel
Softmax 5 × B × h × N² 3 passes over N² Custom CUDA
Attn × V 2 × B × h × N² × d_k O(N² + N×d) cuBLAS batched GEMM
Output projection 2 × B × N × d² O(d²) weights cuBLAS GEMM
Quadratic memory bottleneck

The QKᵀ matmul produces an [N, N] matrix per head. For N=4096, h=32, that's 32 × 4096 × 4096 × 4 bytes = 2GB per layer. This is why FlashAttention was invented.

📊
Layer Normalization
Input: [B, N, d] → Output: [B, N, d]
Memory-bound

What it does

Normalizes activations across the feature dimension (d) for each token independently. Stabilizes training by reducing internal covariate shift.

Mathematical formula
LayerNorm(x) = γ × (x - μ) / √(σ² + ε) + β
μ: mean(x) over d dimension
σ²: variance(x) over d dimension
γ, β: learned scale and shift parameters [d]
ε: small constant for numerical stability (1e-5)

PyTorch implementation

Python PyTorch docs → 
class LayerNorm(nn.Module):
    """Manual implementation to understand the operations"""
    def __init__(self, d_model, eps=1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))   # [d]
        self.beta = nn.Parameter(torch.zeros(d_model))   # [d]
        self.eps = eps

    def forward(self, x):
        # x: [B, N, d]
        # Compute mean and variance over last dimension
        mean = x.mean(dim=-1, keepdim=True)    # [B, N, 1]
        var = x.var(dim=-1, keepdim=True)      # [B, N, 1]

        # Normalize
        x_norm = (x - mean) / torch.sqrt(var + self.eps)  # [B, N, d]

        # Scale and shift
        return self.gamma * x_norm + self.beta  # [B, N, d]

# Or just use PyTorch's optimized version:
layer_norm = nn.LayerNorm(d_model)  # Uses fused CUDA kernel

GPU kernel considerations

LayerNorm is memory-bound because it requires:

  1. Pass 1: Read all d elements to compute mean
  2. Pass 2: Read all d elements again to compute variance
  3. Pass 3: Read all d elements, normalize, write back

Optimization: Fused kernels (like in Apex or TensorRT) compute mean, variance, and normalization in a single pass using online algorithms.

⚡
Feed-Forward Network (FFN)
Input: [B, N, d] → Output: [B, N, d]
Compute-bound

What it does

A position-wise feed-forward network applied identically to each token. This is where most parameters and compute reside (~67% of FLOPs).

Mathematical formula
FFN(x) = Linear₂(Activation(Linear₁(x)))
Linear₁: [d, d_ff] — expands to 4× width
Activation: GELU or SiLU (modern) or ReLU (original)
Linear₂: [d_ff, d] — projects back down

PyTorch implementation

Python GELU docs → 
class FeedForward(nn.Module):
    def __init__(self, d_model, d_ff=None, dropout=0.1):
        super().__init__()
        d_ff = d_ff or 4 * d_model  # Standard: 4× expansion

        self.linear1 = nn.Linear(d_model, d_ff)   # [d, d_ff]
        self.linear2 = nn.Linear(d_ff, d_model)   # [d_ff, d]
        self.activation = nn.GELU()               # Smooth ReLU variant
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        # x: [B, N, d]
        x = self.linear1(x)      # GEMM: [B, N, d] × [d, d_ff] → [B, N, d_ff]
        x = self.activation(x)   # Elementwise GELU
        x = self.dropout(x)
        x = self.linear2(x)      # GEMM: [B, N, d_ff] × [d_ff, d] → [B, N, d]
        return x

# Modern variant: SwiGLU (used in LLaMA, PaLM)
class SwiGLU(nn.Module):
    def __init__(self, d_model, d_ff=None):
        super().__init__()
        d_ff = d_ff or int(2/3 * 4 * d_model)  # Adjusted for gating

        self.w1 = nn.Linear(d_model, d_ff, bias=False)
        self.w2 = nn.Linear(d_ff, d_model, bias=False)
        self.w3 = nn.Linear(d_model, d_ff, bias=False)  # Gate

    def forward(self, x):
        return self.w2(nn.functional.silu(self.w1(x)) * self.w3(x))

Parameter count

For standard FFN with d_ff = 4d:

  • Linear₁: d × d_ff = d × 4d = 4d² parameters
  • Linear₂: d_ff × d = 4d × d = 4d² parameters
  • Total: 8d² parameters per layer

Compare to attention: ~4d² (QKV + output), so FFN has 2× the parameters.

➕
Residual Connection (Add)
Inputs: [B, N, d] + [B, N, d] → Output: [B, N, d]
Memory-bound

What it does

Adds the input of a sub-layer directly to its output, allowing gradients to flow directly through the network. Critical for training deep networks.

Mathematical formula
output = x + SubLayer(LayerNorm(x))
Pre-norm (modern): normalize before sub-layer
Post-norm (original): normalize after add

PyTorch implementation

Python 
class PreNormResidual(nn.Module):
    """Pre-norm style (GPT-2, LLaMA, modern transformers)"""
    def __init__(self, d_model, sublayer):
        super().__init__()
        self.norm = nn.LayerNorm(d_model)
        self.sublayer = sublayer

    def forward(self, x):
        # Residual: x + f(norm(x))
        return x + self.sublayer(self.norm(x))

class PostNormResidual(nn.Module):
    """Post-norm style (original Transformer, BERT)"""
    def __init__(self, d_model, sublayer):
        super().__init__()
        self.norm = nn.LayerNorm(d_model)
        self.sublayer = sublayer

    def forward(self, x):
        # Residual: norm(x + f(x))
        return self.norm(x + self.sublayer(x))

GPU optimization

Residual add is just an elementwise operation, but it's usually fused with LayerNorm to avoid extra memory round-trips:

Conceptual CUDA 
// Instead of:
temp = x + sublayer_output;   // Write to HBM
output = layer_norm(temp);     // Read from HBM, write to HBM

// Fused kernel does:
output = fused_add_layer_norm(x, sublayer_output);  // One read, one write
📈
Softmax
Input: [B, h, N, N] → Output: [B, h, N, N]
Memory-bound

What it does

Converts raw attention scores into a probability distribution over keys. Applied row-wise (each query has a distribution over all keys).

Standard softmax (numerically unstable)
softmax(xᵢ) = exp(xᵢ) / Σⱼ exp(xⱼ)
Safe softmax (subtract max for stability)
softmax(xᵢ) = exp(xᵢ - max(x)) / Σⱼ exp(xⱼ - max(x))
Subtracting max prevents overflow in exp() while giving identical results.

PyTorch implementation

Python PyTorch docs → 
def safe_softmax(x, dim=-1):
    """Manual implementation showing the 3 passes"""
    # Pass 1: Find max (read all elements)
    x_max = x.max(dim=dim, keepdim=True).values

    # Pass 2: Compute exp and sum (read all elements)
    x_exp = torch.exp(x - x_max)
    x_sum = x_exp.sum(dim=dim, keepdim=True)

    # Pass 3: Normalize (read exp, write output)
    return x_exp / x_sum

# PyTorch's version is fused and faster:
output = torch.softmax(scores, dim=-1)

Online softmax (FlashAttention key insight)

Standard softmax needs to see all N elements before outputting any result. Online softmax computes incrementally in a single pass:

Algorithm FlashAttention paper → 
# Online softmax maintains running statistics:
m = -inf       # Running max
l = 0          # Running sum of exp(x - m)

for block in blocks:
    m_new = max(m, block.max())
    l = l * exp(m - m_new) + sum(exp(block - m_new))
    m = m_new

# Final normalization: exp(x - m) / l

Memory is often the limiting factor, especially on edge devices like Jetson.

Edge deployment challenge

A 7B parameter model in FP16 needs ~14GB just for weights. Jetson Orin has 32-64GB unified memory. Quantization (INT8/INT4) is essential for edge deployment.

6 GPU Optimization Techniques

Key optimization strategies for NVIDIA platforms (TensorRT, CUDA, Tensor Cores).

Kernel fusion

🔥
FlashAttention — Fused attention kernel
2-4× speedup, O(N) memory instead of O(N²)

The problem

Standard attention writes the N×N attention matrix to HBM (GPU main memory), then reads it back for softmax and V multiplication. This is extremely memory-bound.

The solution

FlashAttention tiles the computation so everything stays in SRAM (shared memory), never materializing the full N×N matrix.

    enable_mem_efficient=False

    attn_output = F.scaled_dot_product_attention(query, key, value)

"""
Minimal Transformer Implementation
Run: python transformer.py

This implementation prioritizes clarity over efficiency.
For production, use torch.nn.TransformerEncoder or HuggingFace.
"""

import torch
import torch.nn as nn
import torch.nn.functional as F
import math


class MultiHeadAttention(nn.Module):
    """
    Multi-Head Attention mechanism.

    Args:
        d_model: Hidden dimension (embedding size)
        n_heads: Number of attention heads
        dropout: Dropout probability

    Shapes:
        Input:  [B, N, d_model]
        Output: [B, N, d_model]
    """

    def __init__(self, d_model: int, n_heads: int, dropout: float = 0.1):
        super().__init__()
        assert d_model % n_heads == 0, "d_model must be divisible by n_heads"

        self.d_model = d_model
        self.n_heads = n_heads
        self.d_k = d_model // n_heads  # Dimension per head
        self.scale = self.d_k ** -0.5   # 1/sqrt(d_k)

        # Linear projections for Q, K, V
        self.W_q = nn.Linear(d_model, d_model, bias=False)
        self.W_k = nn.Linear(d_model, d_model, bias=False)
        self.W_v = nn.Linear(d_model, d_model, bias=False)
        self.W_o = nn.Linear(d_model, d_model, bias=False)

        self.dropout = nn.Dropout(dropout)

    def forward(self, x: torch.Tensor, mask: torch.Tensor = None) -> torch.Tensor:
        B, N, _ = x.shape

        # Project to Q, K, V
        # Each: [B, N, d_model] @ [d_model, d_model] -> [B, N, d_model]
        Q = self.W_q(x)
        K = self.W_k(x)
        V = self.W_v(x)

        # Reshape for multi-head attention
        # [B, N, d_model] -> [B, N, n_heads, d_k] -> [B, n_heads, N, d_k]
        Q = Q.view(B, N, self.n_heads, self.d_k).transpose(1, 2)
        K = K.view(B, N, self.n_heads, self.d_k).transpose(1, 2)
        V = V.view(B, N, self.n_heads, self.d_k).transpose(1, 2)

        # Compute attention scores
        # [B, h, N, d_k] @ [B, h, d_k, N] -> [B, h, N, N]
        scores = torch.matmul(Q, K.transpose(-2, -1)) * self.scale

        # Apply mask (for causal attention)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))

        # Softmax and dropout
        attn_weights = F.softmax(scores, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # Apply attention to values
        # [B, h, N, N] @ [B, h, N, d_k] -> [B, h, N, d_k]
        context = torch.matmul(attn_weights, V)

        # Reshape back
        # [B, h, N, d_k] -> [B, N, h, d_k] -> [B, N, d_model]
        context = context.transpose(1, 2).contiguous().view(B, N, self.d_model)

        # Output projection
        return self.W_o(context)


class FeedForward(nn.Module):
    """
    Position-wise Feed-Forward Network.

    FFN(x) = Linear2(GELU(Linear1(x)))

    Shapes:
        Input:  [B, N, d_model]
        Output: [B, N, d_model]
    """

    def __init__(self, d_model: int, d_ff: int = None, dropout: float = 0.1):
        super().__init__()
        d_ff = d_ff or 4 * d_model  # Default: 4x expansion

        self.linear1 = nn.Linear(d_model, d_ff)
        self.linear2 = nn.Linear(d_ff, d_model)
        self.dropout = nn.Dropout(dropout)
        self.activation = nn.GELU()

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # [B, N, d] -> [B, N, d_ff] -> [B, N, d]
        x = self.linear1(x)
        x = self.activation(x)
        x = self.dropout(x)
        x = self.linear2(x)
        return x


class TransformerBlock(nn.Module):
    """
    Single Transformer block with pre-normalization.

    Structure:
        x = x + Attention(LayerNorm(x))
        x = x + FFN(LayerNorm(x))
    """

    def __init__(self, d_model: int, n_heads: int, d_ff: int = None, dropout: float = 0.1):
        super().__init__()

        self.norm1 = nn.LayerNorm(d_model)
        self.attn = MultiHeadAttention(d_model, n_heads, dropout)

        self.norm2 = nn.LayerNorm(d_model)
        self.ffn = FeedForward(d_model, d_ff, dropout)

        self.dropout = nn.Dropout(dropout)

    def forward(self, x: torch.Tensor, mask: torch.Tensor = None) -> torch.Tensor:
        # Pre-norm + residual for attention
        x = x + self.dropout(self.attn(self.norm1(x), mask))

        # Pre-norm + residual for FFN
        x = x + self.dropout(self.ffn(self.norm2(x)))

        return x


class Transformer(nn.Module):
    """
    Complete Transformer model.

    Args:
        vocab_size:   Size of vocabulary
        d_model:      Hidden dimension
        n_heads:      Number of attention heads
        n_layers:     Number of transformer blocks
        d_ff:         Feed-forward hidden dimension
        max_seq_len:  Maximum sequence length
        dropout:      Dropout probability
    """

    def __init__(
        self,
        vocab_size: int,
        d_model: int = 512,
        n_heads: int = 8,
        n_layers: int = 6,
        d_ff: int = None,
        max_seq_len: int = 512,
        dropout: float = 0.1
    ):
        super().__init__()

        self.d_model = d_model

        # Token and position embeddings
        self.token_emb = nn.Embedding(vocab_size, d_model)
        self.pos_emb = nn.Embedding(max_seq_len, d_model)
        self.dropout = nn.Dropout(dropout)

        # Transformer blocks
        self.layers = nn.ModuleList([
            TransformerBlock(d_model, n_heads, d_ff, dropout)
            for _ in range(n_layers)
        ])

        # Final layer norm
        self.norm = nn.LayerNorm(d_model)

        # Output projection to vocabulary
        self.output = nn.Linear(d_model, vocab_size, bias=False)

        # Tie weights with token embeddings (common practice)
        self.output.weight = self.token_emb.weight

        # Initialize weights
        self.apply(self._init_weights)

    def _init_weights(self, module):
        if isinstance(module, nn.Linear):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)
            if module.bias is not None:
                torch.nn.init.zeros_(module.bias)
        elif isinstance(module, nn.Embedding):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, N = x.shape

        # Create causal mask
        mask = torch.triu(torch.ones(N, N, device=x.device), diagonal=1).bool()
        mask = ~mask  # Invert: True = attend, False = mask

        # Embeddings
        positions = torch.arange(N, device=x.device).unsqueeze(0)
        x = self.token_emb(x) * math.sqrt(self.d_model)
        x = x + self.pos_emb(positions)
        x = self.dropout(x)

        # Transformer layers
        for layer in self.layers:
            x = layer(x, mask)

        # Output
        x = self.norm(x)
        logits = self.output(x)

        return logits

    def count_parameters(self) -> int:
        """Count total trainable parameters."""
        return sum(p.numel() for p in self.parameters() if p.requires_grad)


def print_model_summary(model: Transformer):
    """Print a summary of the model architecture."""
    print("=" * 60)
    print("TRANSFORMER MODEL SUMMARY")
    print("=" * 60)
    print(f"\nArchitecture:")
    print(f"  - Hidden dimension (d):     {model.d_model}")
    print(f"  - Number of heads (h):      {model.layers[0].attn.n_heads}")
    print(f"  - Head dimension (d_k):     {model.layers[0].attn.d_k}")
    print(f"  - FFN dimension (d_ff):     {model.layers[0].ffn.linear1.out_features}")
    print(f"  - Number of layers (L):     {len(model.layers)}")
    print(f"  - Vocab size (V):           {model.token_emb.num_embeddings}")
    print(f"  - Max sequence length:      {model.pos_emb.num_embeddings}")

    total_params = model.count_parameters()
    print(f"\nTotal parameters: {total_params:,} ({total_params / 1e6:.2f}M)")

    # Parameter breakdown
    print(f"\nParameter breakdown per layer:")
    d = model.d_model
    d_ff = model.layers[0].ffn.linear1.out_features

    qkv = 3 * d * d
    out_proj = d * d
    ffn = 2 * d * d_ff
    layer_norm = 4 * d

    print(f"  - QKV projections: {qkv:,} (3 × d²)")
    print(f"  - Output proj:     {out_proj:,} (d²)")
    print(f"  - FFN:             {ffn:,} (2 × d × d_ff)")
    print(f"  - LayerNorm:       {layer_norm:,} (4 × d)")
    print(f"  - Total/layer:     {qkv + out_proj + ffn + layer_norm:,}")

    print("\n" + "=" * 60)

    """Demonstrate a forward pass with shape annotations."""
    print("\nDEMO: Forward Pass with Shape Tracking")
    print("=" * 60)

    # Model config
    B = 2       # Batch size
    N = 16      # Sequence length
    V = 1000    # Vocab size
    d = 256     # Hidden dimension
    h = 8       # Number of heads
    L = 4       # Number of layers

    print(f"\nConfig: B={B}, N={N}, d={d}, h={h}, L={L}, V={V}")
    print(f"d_k = d/h = {d//h}")
    print(f"d_ff = 4d = {4*d}")

    # Create model
    model = Transformer(
        vocab_size=V,
        d_model=d,
        n_heads=h,
        n_layers=L,
        max_seq_len=512
    )

    # Print summary
    print_model_summary(model)

    # Create dummy input
    x = torch.randint(0, V, (B, N))
    print(f"\nInput shape:  {list(x.shape)} (batch × sequence)")

    # Forward pass
    logits = model(x)
    print(f"Output shape: {list(logits.shape)} (batch × sequence × vocab)")

    # FLOPs estimate
    print(f"\nEstimated FLOPs per forward pass:")
    qkv_flops = 6 * B * N * d * d
    attn_flops = 4 * B * N * N * d
    ffn_flops = 8 * B * N * d * (4 * d)
    total_flops = L * (qkv_flops + attn_flops + ffn_flops)

    print(f"  QKV projection: {qkv_flops * L / 1e6:.2f}M")
    print(f"  Attention:      {attn_flops * L / 1e6:.2f}M")
    print(f"  FFN:            {ffn_flops * L / 1e6:.2f}M")
    print(f"  Total:          {total_flops / 1e6:.2f}M FLOPs")


if __name__ == "__main__":
