💡 The /16 or /14 in ViT Names

ViT-B/16 means patch size P=16. With 224×224 images: (224/16)² = 196 patches.
ViT-H/14 means P=14: (224/14)² = 256 patches. Smaller patches = more tokens = more compute but finer detail.

3. Architecture Blocks (Expandable)

Patch Embedding Layer ▼

Purpose: Turn a 224×224 RGB image into a sequence of 196 patch embeddings, each a 768-dim vector. This is the moment an image "becomes" a token sequence the transformer can consume.

The trick: A Conv2d with kernel_size = stride = patch_size is mathematically identical to "cut image into non-overlapping 16×16 patches, flatten each, apply a linear projection" — but runs as a single fused cuDNN kernel. Same output, better throughput.

class PatchEmbed(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_chans=3, embed_dim=768):
        super().__init__()
        self.num_patches = (img_size // patch_size) ** 2    # (224/16)² = 14² = 196
        # kernel=stride=16 means each output "pixel" sees one non-overlapping 16×16 patch
        self.proj = nn.Conv2d(in_chans, embed_dim,
                              kernel_size=patch_size, stride=patch_size)

    def forward(self, x):
        # x: [B, 3, 224, 224]  — standard NCHW image tensor
        x = self.proj(x)                    # Conv2d → [B, 768, 14, 14]
        x = x.flatten(2).transpose(1, 2)    # [B, 768, 196] → [B, 196, 768]
        return x

Parameters: 3 × 16 × 16 × 768 + 768 (bias) = 590,592 — just 0.7% of ViT-B's 86M. Patch embed is tiny; the transformer layers dominate.

FLOPs: 2 × (224 × 224) × (3 × 16 × 16) × 768 / (16 × 16) = 2 × 196 × 768 × 768 ≈ 0.23 GFLOPs. Convs of this shape run well on Tensor Cores — treat as a plain GEMM with M=196, N=768, K=768.

Position + CLS Token ▼

Purpose: Add two things to the patch sequence — a learnable [CLS] token that will collect global image info through self-attention, and position embeddings that tell the model where each patch came from in the 2D grid.

Why a CLS token at all? After L attention layers, we need a single vector that represents the whole image for classification. We could average all 196 patch tokens instead (global average pooling), and some ViT variants do — but the learnable CLS token works slightly better because it learns what to attend to.

# In ViT __init__:
self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))          # [1, 1, 768] — one learnable vec
self.pos_embed = nn.Parameter(torch.zeros(1, num_patches + 1, embed_dim))  # [1, 197, 768] — one per slot

# In forward:
def forward(self, x):
    B = x.shape[0]
    x = self.patch_embed(x)                          # [B, 3, 224, 224] → [B, 196, 768]

    # Prepend the CLS token — same vector broadcast across the batch
    cls_tokens = self.cls_token.expand(B, -1, -1)   # [B, 1, 768]
    x = torch.cat([cls_tokens, x], dim=1)          # [B, 197, 768]

    # Add position info — broadcast over the batch dim
    x = x + self.pos_embed                            # [B, 197, 768]
    return x

Note: Position embeddings are 1D indices (0–196) but learn 2D spatial structure during training — adjacent patches end up with similar embeddings. Some newer models (DeiT, Swin) use 2D or relative position embeddings instead.

Multi-Head Self-Attention (Same as LLM) ▼

Purpose: Let every patch look at every other patch and decide what's relevant. After this layer, each patch's embedding blends information from across the whole image.

How it splits the work: The 768-dim embedding is sliced into h=12 heads of 64 dims each. Each head computes its own Q·Kᵀ score matrix, applies softmax, and weights V independently. Heads learn different attention patterns (edges, textures, object parts) and are concatenated at the end.

class Attention(nn.Module):
    def __init__(self, dim=768, num_heads=12):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = dim // num_heads         # 64 — each head's slice of embedding
        self.scale = self.head_dim ** -0.5    # 1/sqrt(64)=0.125, keeps softmax stable

        # One Linear for Q,K,V is faster than three — a single batched GEMM
        self.qkv = nn.Linear(dim, dim * 3)        # weight: [768, 2304]
        self.proj = nn.Linear(dim, dim)            # output projection W_o

    def forward(self, x):
        B, N, C = x.shape                          # [B, 197, 768] — batch, tokens, embed

        # Project and split into Q, K, V for all heads in one shot
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, self.head_dim)
        qkv = qkv.permute(2, 0, 3, 1, 4)    # [3, B, 12, 197, 64] → unbind QKV
        q, k, v = qkv[0], qkv[1], qkv[2]       # each: [B, 12, 197, 64]

        # Score matrix: how much does each query look at each key?
        # NO CAUSAL MASK in ViT — every patch sees every patch (bidirectional)
        attn = (q @ k.transpose(-2, -1)) * self.scale  # [B, 12, 197, 197]
        attn = attn.softmax(dim=-1)                       # row-wise → each query sums to 1

        # Weighted sum of values, then merge heads back into one 768-dim vector
        x = (attn @ v).transpose(1, 2).reshape(B, N, C)    # [B, 197, 768]
        x = self.proj(x)                                        # final mix across heads
        return x

Key difference from decoder-only LLM: No causal mask. In an LLM, token t can only attend to tokens 0..t (future is masked out for training); in ViT, patch 0 can attend to patch 196 and vice versa. This is why ViT is an encoder, not a decoder.

GPU view: The two matmuls (Q·Kᵀ and attn·V) are batched GEMMs on Tensor Cores. The softmax in between is memory-bound. FlashAttention fuses all three into one kernel, never writing the [B, 12, 197, 197] score matrix to HBM — for ViT-B this is modest (~60 MB per batch at FP16) but matters at scale.

Feed-Forward Network (MLP) ▼

Purpose: Non-linear transformation applied independently to each token. Attention mixes between tokens; the MLP mixes within each token's embedding.

Why 4× expansion: The hidden layer is 4× wider than the input (768 → 3072 → 768). This gives the network capacity to learn rich features per token. The 4× ratio is empirical — changing it rarely helps.

class MLP(nn.Module):
    def __init__(self, dim=768, mlp_ratio=4.0):
        super().__init__()
        hidden_dim = int(dim * mlp_ratio)        # 3072 for ViT-B
        self.fc1 = nn.Linear(dim, hidden_dim)     # expand: [768 → 3072]
        self.act = nn.GELU()                      # smooth activation, not ReLU
        self.fc2 = nn.Linear(hidden_dim, dim)     # contract: [3072 → 768]

    def forward(self, x):
        x = self.fc1(x)   # [B, 197, 768] → [B, 197, 3072]
        x = self.act(x)   # element-wise GELU
        x = self.fc2(x)   # [B, 197, 3072] → [B, 197, 768]
        return x

FLOPs per layer: 2 × B × N × (d · d_ff) × 2 passes = 2 × 1 × 197 × 768 × 3072 × 2 ≈ 1.85 GFLOPs (ViT-B, batch 1). Across 12 layers: ~22 GFLOPs, which is about 67% of the model's total compute — MLPs, not attention, dominate ViT's FLOP budget.

GPU view: Both linears are pure GEMMs on Tensor Cores — compute-bound at reasonable batch sizes. GELU is element-wise and memory-bound; it's typically fused with fc1 in production (e.g., cuBLASLt epilogue or TensorRT plugin) to avoid the round-trip to HBM.

Classification Head ▼

Purpose: Take the CLS token's final embedding (after L layers of attention) and map it to class logits. The CLS token has aggregated information from every patch by this point — it's the image's global representation.

class ViT(nn.Module):
    def __init__(self, ..., num_classes=1000):
        # ... patch embed, encoder blocks ...
        self.norm = nn.LayerNorm(embed_dim)
        self.head = nn.Linear(embed_dim, num_classes)    # 768 → 1000 (ImageNet)

    def forward(self, x):
        B = x.shape[0]
        x = self.patch_embed(x)                                   # [B, 196, 768]
        x = torch.cat([self.cls_token.expand(B, -1, -1), x], dim=1)  # [B, 197, 768]
        x = x + self.pos_embed                                    # [B, 197, 768]

        for block in self.blocks:
            x = block(x)                     # [B, 197, 768] — shape unchanged

        x = self.norm(x)                     # final LayerNorm before head
        cls_token = x[:, 0]                  # [B, 768] — grab just the CLS slot
        return self.head(cls_token)        # [B, 1000] — class logits

Alternative: Some variants (DeiT-III, later ViT papers) use global average pooling over all patch tokens instead of the CLS token. Marginal accuracy tradeoff; CLS remains standard.

5. ViT vs CNN: When to Use What

Aspect CNN (ResNet, EfficientNet) ViT
Inductive Bias Strong: locality, translation equivariance Weak: learns everything from data
Data Efficiency Better with small datasets (<1M images) Needs large data (10M+ images) or pretraining
Scaling Saturates earlier Better scaling laws, continues to improve
Global Context Requires deep networks (large receptive field) From layer 1 (self-attention)
Throughput (images/sec) Generally faster (optimized kernels) Slower but improving (FlashAttention)
Transfer Learning Good Excellent (especially with large pretraining)
💡 Practical Guidance

Use CNN when: Limited data, edge deployment, real-time requirements

Use ViT when: Large pretraining available, multimodal applications, best accuracy needed

Hybrid approaches: ConvNeXt (CNN with transformer tricks), Early Conv + ViT (best of both)

6. GPU Optimization Techniques

ViT-Specific Optimizations

Technique Benefit Implementation
FlashAttention 2-4× faster attention, O(N) memory torch.nn.functional.scaled_dot_product_attention
Fused Patch Embedding Conv2d + reshape in one kernel TensorRT, custom CUDA kernels
INT8 Quantization 2× throughput, 2× memory reduction torch.quantization, TensorRT
FP16/BF16 2× memory, faster matmuls torch.cuda.amp, model.half()
Operator Fusion Reduce memory bandwidth LayerNorm + Dropout fused, QKV single matmul
Dynamic Resolution Flexible input sizes, interpolate pos_embed F.interpolate on position embeddings
⚠️ ViT Memory Considerations

Unlike LLMs, ViT attention memory is usually NOT the bottleneck:

• N=197 patches → attention scores: 197 × 197 × 12 × 4 = 1.8 MB per image

• Compare to LLM with N=4096: 4096 × 4096 × 32 × 4 = 2 GB!

• FFN activations dominate: 197 × 3072 × 12 × 4 = 29 MB per image

7. Reference Implementation

"""
Vision Transformer (ViT) Reference Implementation
Optimized for clarity and inference understanding
"""
import torch
import torch.nn as nn
import torch.nn.functional as F
from typing import Optional

class PatchEmbed(nn.Module):
    """Convert image to patch embeddings using Conv2d."""
    def __init__(self, img_size=224, patch_size=16, in_chans=3, embed_dim=768):
        super().__init__()
        self.img_size = img_size
        self.patch_size = patch_size
        self.num_patches = (img_size // patch_size) ** 2
        self.proj = nn.Conv2d(in_chans, embed_dim, kernel_size=patch_size, stride=patch_size)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        # x: [B, 3, H, W] -> [B, num_patches, embed_dim]
        x = self.proj(x)  # [B, embed_dim, H/P, W/P]
        x = x.flatten(2).transpose(1, 2)  # [B, num_patches, embed_dim]
        return x

class Attention(nn.Module):
    """Multi-head self-attention (no causal mask)."""
    def __init__(self, dim=768, num_heads=12, qkv_bias=True):
        super().__init__()
        self.num_heads = num_heads
        self.head_dim = dim // num_heads
        self.scale = self.head_dim ** -0.5

        self.qkv = nn.Linear(dim, dim * 3, bias=qkv_bias)
        self.proj = nn.Linear(dim, dim)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B, N, C = x.shape
        qkv = self.qkv(x).reshape(B, N, 3, self.num_heads, self.head_dim).permute(2, 0, 3, 1, 4)
        q, k, v = qkv.unbind(0)

        # Use PyTorch 2.0+ scaled_dot_product_attention for efficiency
        x = F.scaled_dot_product_attention(q, k, v)  # [B, h, N, d_k]
        x = x.transpose(1, 2).reshape(B, N, C)
        return self.proj(x)

class MLP(nn.Module):
    """Feed-forward network with GELU activation."""
    def __init__(self, dim=768, mlp_ratio=4.0):
        super().__init__()
        hidden = int(dim * mlp_ratio)
        self.fc1 = nn.Linear(dim, hidden)
        self.fc2 = nn.Linear(hidden, dim)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.fc2(F.gelu(self.fc1(x)))

class Block(nn.Module):
    """Transformer encoder block with pre-norm."""
    def __init__(self, dim=768, num_heads=12, mlp_ratio=4.0):
        super().__init__()
        self.norm1 = nn.LayerNorm(dim)
        self.attn = Attention(dim, num_heads)
        self.norm2 = nn.LayerNorm(dim)
        self.mlp = MLP(dim, mlp_ratio)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = x + self.attn(self.norm1(x))  # Pre-norm + residual
        x = x + self.mlp(self.norm2(x))
        return x

class ViT(nn.Module):
    """Vision Transformer for image classification."""
    def __init__(
        self,
        img_size=224,
        patch_size=16,
        in_chans=3,
        num_classes=1000,
        embed_dim=768,
        depth=12,
        num_heads=12,
        mlp_ratio=4.0,
    ):
        super().__init__()
        self.patch_embed = PatchEmbed(img_size, patch_size, in_chans, embed_dim)
        num_patches = self.patch_embed.num_patches

        self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))
        self.pos_embed = nn.Parameter(torch.zeros(1, num_patches + 1, embed_dim))

        self.blocks = nn.ModuleList([
            Block(embed_dim, num_heads, mlp_ratio) for _ in range(depth)
        ])
        self.norm = nn.LayerNorm(embed_dim)
        self.head = nn.Linear(embed_dim, num_classes)

        self._init_weights()

    def _init_weights(self):
        nn.init.trunc_normal_(self.pos_embed, std=0.02)
        nn.init.trunc_normal_(self.cls_token, std=0.02)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        B = x.shape[0]
        x = self.patch_embed(x)  # [B, 196, 768]

        cls_tokens = self.cls_token.expand(B, -1, -1)
        x = torch.cat([cls_tokens, x], dim=1)  # [B, 197, 768]
        x = x + self.pos_embed

        for block in self.blocks:
            x = block(x)

        x = self.norm(x)
        return self.head(x[:, 0])  # [B, num_classes]

# Model configurations
def vit_base_patch16():
    return ViT(patch_size=16, embed_dim=768, depth=12, num_heads=12)

def vit_large_patch16():
    return ViT(patch_size=16, embed_dim=1024, depth=24, num_heads=16)

def vit_huge_patch14():
    return ViT(patch_size=14, embed_dim=1280, depth=32, num_heads=16)

if __name__ == "__main__":
    model = vit_base_patch16()
    print(f"ViT-B/16 Parameters: {sum(p.numel() for p in model.parameters()):,}")

    x = torch.randn(1, 3, 224, 224)
    with torch.no_grad():
        out = model(x)
    print(f"Input: {x.shape} -> Output: {out.shape}")
